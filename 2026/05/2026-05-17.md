###### 컨텍스트 전파

**컨텍스트 주입**

아웃바운드 호출을 만들 때는 주로 다운스트림 서비스에 컨텍스트를 전파하고자 할 것이다. 이 경우에는 외부 호출을 추적할 스팬을 생성하고 `Propagator` API를 사용해 컨텍스트를 메시지에 주입한다. 비동기 처리를 위한 메시지 생성처럼 컨텍스트를 추입하고자 하는 다른 경우가 있을 수도 있다. 다음 Java 예시는 컨텍스트를 전파하는 방법을 보여준다. 더 많은 예시는 [Java에서 컨텍스트 주입][otel-languages-java-instrumentation-context-propagation]을 참고하라.

```java
Span span = tracer.spanBuilder("send")
            .setSpanKind(SpanKind.CLIENT)
            .startSpan();

// make span active so any nested telemetry is correlated
// even network calls might have nested layers of spans, logs or events
try (Scope unused = span.makeCurrent()) {
  // inject the context
  propagator.inject(Context.current(), transportLayer, setter);
  send();
} catch (Exception e) {
  span.recordException(e);
  span.setStatus(StatusCode.ERROR);
  throw e;
} finally {
  span.end();
}
```

컨텍스트를 전파할 필요가 없는 몇가지 예외가 있다:

- 메타데이터를 지원하지 않거나 알려지지 않은 필드를 금지하는 다운스트림 서비스.
- 연관 프로토콜을 정의하지 않는 다운스트림 서비스. 이후 버전에서 컨텍스트 전파 지원을 추가하는 것을 고려하라.
- 커스텀 연관 프로토콜을 지원하는 다운스트림 서비스.
  - 커스텀 전파자를 사용하는 최선의 노력은 호환되는 경우 OpenTelemetry 트레이스 컨텍스트를 사용하거나 스팬에 대한 커스텀 연관 ID를 생성하고 설정하는 것이다.
