###### 컨텍스트 전파

**컨텍스트 추출**

웹 프레임워크나 메시징 컨슈머와 같이 라이브러리나 업스트림 호출을 수신하는 서비스를 작업하고 있다면 인입 요청이나 메시지로부터 컨텍스트를 추출하라. OpenTelemetry는 구체적인 전파 표준을 숨기고 통신에서 트레이스 `Context`를 읽는 `Propagator` API를 제공한다. 단일 응답의 경우 라이브러리가 생성하는 새 스팬의 부모가 되는 하나의 컨텍스트가 있다.

스팬을 만들고 나면 새 스팬을 활성으로 만들어 새 트레이스 컨텍스트를 애플리케이션 코드(콜백 또는 핸들러)에 추가하라. 가능하면 이를 명시적으로 하라. 다음 java 예시는 트레이스 컨텍스트를 추가하고 스팬을 활성화하는 방법을 설명한다. 자세한 내용은 [Java에서 컨텍스트 추출][otel-languages-java-api-context-propagators]을 참고하라.

```java
// extract the context
Context extractedContext = propagator.extract(Context.current(), httpExchange, getter);
Span span = tracer.spanBuilder("receive")
            .setSpanKind(SpanKind.SERVER)
            .setParent(extractedContext)
            .startSpan();

// make span active so any nested telemetry is correlated
try (Scope unused = span.makeCurrent()) {
  userCode();
} catch (Exception e) {
  span.recordException(e);
  span.setStatus(StatusCode.ERROR);
  throw e;
} finally {
  span.end();
}
```

메시징 시스템의 경우, 한 번에 하나 이상의 메시지를 수신할 수 있다. 수신한 메시지는 생성하는 스팬의 링크가 된다. 자세한 내용은 [메시징 컨벤션][otel-specs-semconv-messaging-messaging-spans]을 참고하라.
