**성능**

OpenTelemetry API는 no-op이며 애플리케이션에 SDK가 없을 때는 성능이 매우 뛰어나다. OpenTelemetry SDK가 구성되면 [바운드된 리소스를 소비한다][otel-specs-performance].

실제 애플리케이션, 특히 대규모에서는 헤드 기반 샘플링이 자주 구성된다. 샘플링된 스팬은 입수 가능(affordable)하고 스팬을 기록중인지 확인해 속성을 채우는 과정에서 추가적인 할당과 비쌀 수 있는 계산을 피할 수 있다. 다음 Java 예시는 샘플링을 위한 속성을 제공하고 스팬을 기록중인지 확인하는 것을 보여준다:

```java
// some attributes are important for sampling, they should be provided at creation time
Span span = tracer.spanBuilder(String.format("SELECT %s.%s", dbName, collectionName))
        .setSpanKind(SpanKind.CLIENT)
        .setAttribute("db.name", dbName)
        ...
        .startSpan();

// other attributes, especially those that are expensive to calculate
// should be added if span is recording
if (span.isRecording()) {
    span.setAttribute("db.statement", sanitize(query.statement()))
}
```

[otel-specs-performance]: https://opentelemetry.io/docs/specs/otel/performance/
