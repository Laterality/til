# 2026-03-03

## OpenTelemtry

### 개념

#### 컨텍스트 전파

##### 보안 모범 사례

전파는 보안상 고려가 필요한 서비스 경계 간 데이터 송수신이 발생한다.

###### 외부 서비스

서비스가 (소유하거나 신뢰하지 않는)외부 서비스와 상호작용할 때는 다음을 고려하라:

- **인입 컨텍스트**: 외부 소스로부터 컨텍스트를 수락할 때는 주의하라. 악의적인 행위자가 위조된 트레이스 헤더를 보내 트레이싱 데이터를 조작하거나 컨텍스트 파싱의 취약점을 악용할 수 있다. 신뢰할 수 없는 소스로부터의 인입 컨텍스트는 무시하거나 소독(sanitize)하는 것이 좋다.
- **반출 컨텍스트**: 외부 서비스로 무엇을 전파하는지 유의하라. 내부 트레이스 IDs, 스팬 IDs, 배기지 항목은 내부 아키텍처나 비지니스 로직에 관한 민감한 정보를 드러낼 수 있다. 전파자가 외부 혹은 외부와 맞닿은 엔드포인트로는 컨텍스트를 보내지 않도록 구성하는 것이 좋다.


###### 배기지(Baggage)

[배기지][otel-concepts-signals-baggage]는 임의의 키-값 쌍을 전파할 수 있게 한다. 이 데이터는 서비스 경계를 넘어 전파되기 때문에, 로깅되거나 신뢰할 수 없는 다운스트림 서비스로 보내질 수 있으므로 민감한 정보(사용자 크리덴셜, API 키, PII)를 배기지에 넣는 것을 피하라.

[otel-concepts-signals-baggage]: https://opentelemetry.io/docs/concepts/signals/baggage/
