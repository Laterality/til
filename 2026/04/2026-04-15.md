###### OpenTelemetry 로깅 컴포넌트

**로그 레코드**

로그 레코드는 이벤트의 기록을 나타낸다. OpenTelemetry에서 로그 레코드는 두 종류의 필드를 갖는다:

- 특정 타입과 의미를 가진 명명된 최상위 필드
- 임의의 값과 타입을 가진 리소스와 속성 필드

최상위 필드:

| 필드명                | 설명                                     |
| --------------------- | ---------------------------------------- |
| Timestamp             | 이벤트가 발생한 시간.                    |
| ObservedTimestamp     | 이벤트가 관측된 시간.                    |
| TraceId               | 요청 트레이스 ID.                        |
| SpanId                | 요청 스팬 ID.                            |
| TraceFlag             | W3C 트레이스 플래그.                     |
| SeverityText          | (로그 레벨로도 알려져 있는)심각도 텍스트 |
| SeverityNumber        | 심각도의 수치 값.                        |
| Body                  | 로그 레코드의 본문.                      |
| Resource              | 로그의 소스를 설명한다.                  |
| InstrumentiationScope | 로그를 방출한 스코프를 설명한다.         |
| Attributes            | 이벤트에 관한 추가 정보.                 |

로그 레코드와 로그 필드에 관한 더 자세한 내용은 [로그 데이터 모델][otel-specs-otel-logs-data-model]을 참고하라.

[otel-specs-otel-logs-data-model]: https://opentelemetry.io/docs/specs/otel/logs/data-model/
