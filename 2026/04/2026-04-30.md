##### 추가적인 OpenTelemetry 혜택

###### 제로 코드

[옵스][otel-gettings-started-ops](ops)로서 여러분은 소스를 수정할 필요 없이 하나 이상의 애플리케이션에 관측가능성을 더하고 싶을지도 모른다. OpenTelemetry는 여러분이 [코드 기반 계측][otel-concepts-instrumentation-code-based]을 위한 OpenTelemtry API & SDK를 사용할 필요 없이 빠르게 약간의 관측가능성을 얻을 수 있게 한다.

제로 코드 계측은 주로 에이전트나 에이전트와 비슷한 설치 방식으로 애플리케이션에 OpenTelemetry API와 SDK 기능을 더한다. 바이트코드 조작, 몽키 패칭, 또는 eBPF 등 애플리케이션에 OpenTelemetry API와 SDK 호출을 주입하는 방법은 특정 메커니즘은 언어마다 다를 수 있다.

주로, 제로 코드 계측은 사용중인 라이브러리에 대한 계측을 추가한다. 즉, 요청과 응답, 데이터베이스 호출, 메시지 큐 호출과 같은 것들이 계측된다. 하지만 여러분의 애플리케이션 코드는 보통 계측되지 않는다. 여러분의 코드를 계측하려면 [코드 기반 계측][otel-concepts-instrumentation-code-based]을 사용해야 할 것이다.

게다가, 제로 코드 계측은 [게측 라이브러리][otel-concepts-instrumentation-libraries]와 [익스포터][otel-concepts-components-exoprters]가 로드되도록 구성하게 한다.

환경 변수와 시스템 프로퍼티나 초기화 메서드에 인자 전달 등 기타 언어 특화 메커니즘을 통해 제로 코드 계측을 구성할 수 있다. 시작하기 위해서는 선택한 관측가능성 백엔드에서 서비스를 식별하기 위한 서비스 이름만 구성하면 된다.

다음을 포함해 다른 구성 옵션들을 사용할 수 있다:

- 데이터 소스 특화 구성
- 익스포터 구성
- 전파자 구성
- 리소스 구성

다음 언어들에 대해 추가 계측을 사용할 수 있다:

- [.NET][otel-zero-code-dotnet]
- [Go][otel-zero-code-go]
- [Java][otel-zero-code-java]
- [JavaScript][otel-zero-code-js]
- [PHP][otel-zero-code-php]
- [Python][otel-zero-code-python]
