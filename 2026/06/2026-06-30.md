## 개요

### Kubernetes의 객체

#### Kubernetes 객체 이해하기

*KUbernetes 객체*는 Kubernetes 시스템의 영속적인 엔티티다. Kubernetes는 이 엔티티들을 사용해 클러스터의 상태를 나타낸다. 구체적으로는 다음을 기술할 수 있다:

- (어느 노드에서)컨테이너화된 애플리케이션이 실행중인 것
- 애플리케이션이 사용할 수 있는 리소스
- 재시작 정책, 업그레이드, 장애 허용도 등 애플리케이션 동작과 관련된 정책들

하나의 Kubernetes 객체는 "의도의 기록"이다. 객체를 만들면 Kubernetes 시스템은 끊임없이 객체가 존재하는지 확인한다. 객체를 생성함으로써 클러스터의 워크로드가 어떤 모습이길 원하는지 Kubernetes 시스템에 전달한다. 이것이 클러스터의 *원하는 상태*(desired state)다.

생성, 수정 삭제 등 Kubernetes 객체를 활용하려면 [Kubernetes API][k8s-concepts-kubernetes-api]를 사용해야 한다. 예를 들어, `kubectl` 커맨드 라인 인터페이스를 사용하면 CLI가 필요한 Kubernetes API 호출을 생성한다. 또한 [클라이언트 라이브러리][k8s-reference-using-api-client-libraries] 중 하나를 여러분의 프로그램에서 사용해 Kubernetes API를 직접 사용할 수도 있다.

[k8s-reference-using-api-client-libraries]: https://kubernetes.io/docs/reference/using-api/client-libraries/
