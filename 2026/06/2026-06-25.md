## 개요

### Kubernetes 구성요소

#### 핵심 구성요소

하나의 Kubernetes 클러스터는 컨트롤 플레인과 하나 이상의 워커 노드로 구성된다. 다음은 주요 구성요소의 대략적인 개요다:

##### 컨트롤 플레인 구성요소

클러스터의 전반적인 상태를 관리한다:

- [kube-apiserver][k8s-v1-35-concepts-architecture-kube-apiserver]: Kubernetes HTTP API를 노출하는 핵심 구성요소 서버.
- [etcd][k8s-v1-35-concepts-architecture-etcd]: 일관성과 고가용성을 제공하는 모든 API 서버 데이터를 위한 키 값 저장소.
- [kube-scheduler][k8s-v1-35-concepts-architecture-kube-scheduler]: 노드에 아직 바운드되지 않은 파드를 찾고 각 파드를 적합한 노드에 할당한다.
- [kube-controller-manager][k8s-v1-35-concepts-architecture-kube-controller-manager]: Kubernetes API 동작을 구현하도록 [컨트롤러][k8s-v1-35-concepts-architecture-controller]를 실행한다.
- [cloud-controller-manager][k8s-v1-35-concepts-architecture-cloud-controller-manager](선택사항): 기반 클라우드 제공자(들)과 통합한다.
