##### Kubernetes 객체 기술하기

Kubernetes에서 객체를 생성할 때는 객체가 원하는 상태를 기술하는 객체 스펙뿐만아니라 (이름과 같은)객체 자체에 대한 기본 정보도 제공해야 한다. (직접 혹은 `kubectl`을 통해)Kubernetes API를 사용해 `object`를 생성할 때는, API 요청이 요청 본문에 그 내용을 JSON으로 포함해야 한다. 가장 자주 *매니페스트*로 알려진 파일에 `kubectl`에 필요한 정보를 제공한다. 관례에 따라 매니페스트는 YAML이다(JSON 형식도 사용할 수 있다). `kubectl`과 같은 도구들은 HTTP를 통한 API 요청을 만들 때 매니페스트에서 JSON이나 다른 지원되는 직렬화 형식으로 변환한다.

다음은 Kubernetes 디플로이먼트의 필수 필드와 객체 스펙을 보여주는 예시 매니페스트다:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  selector:
    matchLabels:
      app: nginx
  replicas: 2 # tells deployment to run 2 pods matching the template
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80
```

위에 있는 것과 같은 매니페스트 파일을 사용해 디플로이먼트를 생성하는 한 가지 방법은 `kubectl` 커맨드 라인 인터페이스에서 `.yaml` 파일을 인자로 넘겨 [`kubectl apply`][k8s-kubectl-command-apply] 커맨드를 사용하는 것이다. 다음은 예시다:

```bash
kubectl apply -f https://k8s.io/examples/application/deployment.yaml
```

출력은 아래와 비슷하다:

```plaintext
deployment.apps/nginx-deployment created
```

[k8s-kubectl-command-apply]: https://kubernetes.io/docs/reference/generated/kubectl/kubectl-commands#apply
