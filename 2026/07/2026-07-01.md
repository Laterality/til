## 개요

### Kubernetes의 객체

#### Kubernetes 객체 이해하기

##### 객체 스펙과 상태

거의 대부분의 Kubernetes 객체는 객체의 구성을 지배하는 두 개의 중첩된 객체 필드 `spec`과 `status`를 갖는다. `spec`을 갖는 객체의 경우, 객체를 생성할 때 리소스가 갖기를 원하는 특성의 설명을 제공한다. 즉, *원하는 상태*(desired state)다.

`status`는 Kubernetes 시스템과 그 구성요소들이 제공하고 갱신해 객체의 *현재 상태*를 기술한다. Kubernetes 컨트롤 플레인은 지속적이고 적극적으로 모든 객체의 실제 상태가 여러분이 제공한 원하는 상태와 일치하도록 관리한다.

예를 들어, Kubernetes에서 디플로이먼트는 클러스터에서 실행중인 애플리케이션을 나타낸다. 디플로이먼트를 생성할 때는 디플로이먼트 `spec`을 설정해 세 개의 애플리케이션 복제본이 실행되길 원한다고 기술할 수 있다. Kubernetes 시스템은 디플로이먼트 스펙을 읽고 원하는 애플리케이션의 인스턴스 세 개를 시작해 상태가 스펙과 일치하도록 갱신한다. 인스턴스 중 하나라도 실패(상태 변화)하면 Kubernetes 시스템은 스펙과의 차이에 응답해 정정을 수행한다. 이 경우에는 새 대체 인스턴스를 시작한다.

객체 스펙, 상태 및 메타데이터에 관한 더 자세한 내용은 [Kubernetes API 컨벤션][github-kubernetes-community-sig-architecture-api-conventions]을 참고하라.
