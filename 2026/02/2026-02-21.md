## OpenTelemtry

### 개념

#### 관측가능성 프라이머(primer)

##### 스팬

**스팬**은 하나의 작업 또는 연산 단위를 나타낸다. 스팬은 요청이 만들어내는 특정 연산을 추적하고 연산이 실행되는 동안 어떤 일이 벌어지는지 그려낸다.

스팬은 이름, 시간 관련 데이터, [구조화된 로그 메시지][otel-concepts-signals-traces-span-events], 그리고 [다른 메타데이터(즉, 속성)][otel-concepts-signals-traces-attributes]을 가져 자신이 추적하는 연산에 대한 정보를 제공한다.

###### 스팬 속성

스팬 속성은 스팬에 추가되는 메타데이터다.

다음 표는 스팬 속성의 예시다:

| Key                       | Value                                                                            |
| ------------------------- | -------------------------------------------------------------------------------- |
| http.request.method       | "GET"                                                                            |
| network.protocol.version  | "1.1"                                                                            |
| url.path                  | "/webshop/articles/4"                                                            |
| url.query                 | "?s=1"                                                                           |
| server.address            | "example.com"                                                                    |
| server.port               | 8080                                                                             |
| url.scheme                | "https"                                                                          |
| http.route                | "/webshop/articles/:article_id"                                                  |
| http.response.status_code | 200                                                                              |
| client.address            | "192.0.2.4"                                                                      |
| client.socket.address     | "192.0.2.5" (the client goes through a proxy)                                    |
| user_agent.original       | "Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:72.0) Gecko/20100101 Firefox/72.0" |

스팬과 스팬이 어떻게 OpenTelemetry와 연관되는지는 [스팬][otel-concepts-signals-trace-spans]을 참고하라.
