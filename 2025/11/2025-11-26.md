## Envoy

### 아키텍처 개요

#### 기타 프로토콜

##### gRPC

[gRPC][grpc]는 Google의 RPC 프레임워크다. 프로토콜 버퍼를 기반 직렬화/IDL 형식으로 사용한다. 전송 계층에서 요청/응답 멀티플렉싱에 HTTP/2 이상을 사용한다. Envoy는 전송 계층뿐만 아니라 응용 계층에서도 gRPC에 대한 일급 지원을 갖고 있다.

* gRPC는 요청 상태를 나르는 데 트레일러(trailer)를 사용한다. Envoy는 트레일러를 올바르게 지원하는 소수의 HTTP 프록시 중 하나이며 따라서 gRPC 요청과 응답을 전송할 수 있는 소수의 프록시 중 하나이다.
* 일부 언어에서 gRPC 런타임이 비약하다. gRPC를 더 많은 언어에 사용하는 데 도움이 되는 필터의 소개는 [아래][arch-other-protocols-grpc-bridging]를 참고하라.
* gRPC-Web은 gRPC-Web 클라이언트에 HTTP/1.1을 통해 Envoy로 요청을 보내고 gRPC 서버로 프록시되도록 허용하는 [필터][config-http-http-filters-grpc-web]에 의해 지원된다. 아직 개발이 진행중이며 gRPC [브릿지 필터][config-http-http-filters-grpc-http-1-1-bridge]의 다음 세대가 될 것으로 예상된다.
* gRPC-JSON 트랜스코더는 RESTful JSON API 클라이언트가 HTTP를 통해 Envoy로 요청을 보내고 gRPC 서비스로 프록시되도록 허용하는 [필터][config-http-http-filters-grpc-json-transcoder]에 의해 지원된다.
* gRPC-JSON 리버스 트랜스코더는 gRPC 클라이언트가 Envoy로 요청을 보내고 RESTful JSON API 서비스로 프록시되도록 허용하는 [필터][config-http-http-filters-grpc-json-reverse-transcoder]에 의해 지원된다.
