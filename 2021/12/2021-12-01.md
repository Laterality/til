## 2021. 12. 01.

### RFC7519 - JSON Web Token

#### 7. JWT 생성과 유효성 검증(3)

##### 7.3. 문자열 비교 규칙

JWT 처리에는 알려는 문자열을 JSON 객체의 멤버과 값과 비교하는 일이 필요하다. 예를 들어, 어떤 알고리즘인지 확인할 때, 일치하는 파라미터 이름이 있다면 JOSE 헤더의 멤버 이름에 대해 유니코드 문자열 인코딩 "alg"를 확인할 것이다.

멤버 이름 비교에 대한 JSON 규칙은 RFC 7159의 Section 8.3에 기술되어있다. 문자열 비교 연산은 같은지 다른지만 비교하므로, 알려진 문자열과 멤버 이름과 멤버 값에도 같은 규칙을 사용할 수 있다.

이러한 비교 규칙은 멤버의 정의에서 멤버 값에 대해 명시적으로 다른 비교 규칙을 사용하는 것이 아닌 이상 모든 JSON 문자열에 **사용해야 한다**. 이 명세에서는 "typ"와 "cty" 멤버 값만 이 비교 규칙을 사용하지 않는다.

일부 애플리케이션은 DNS 이름을 "iss" (issuer) 클레임 값의 일부로 포함하는 경우와 같이 대소문자를 구분하는 값에 대소문자를 구분하지 않는 정보를 포함시킬 수도 있다. 이러한 경우에는 둘 이상의 당사자(party)가 동일한 값을 만들어내는 경우 이를 비교할 수 있도록 소문자로 바꾸는 등, 애플리케이션이 대소문자를 구분하지 않는 부분을 표현하는 데 사용할 정규화된 형식을 컨벤션으로 정의해야 할 수 있다. 하지만 다른 모든 당사자가 값이 무엇인지와는 관계없이 생성된 값을 비교하지 않고 값을 그대로 사용한다면 생성자가 사용하는 대소문자는 문제가 되지 않을 것이다.
